# 域名可达性检测

> **使用声明**：本工具仅用于站长检测**自有域名**在各地区的网络可达性，请勿用于非授权域名的批量扫描。检测数据由第三方公开平台提供，请合理控制调用频率，遵守对应平台服务条款。

## 触发方式
- **触发词**: `/check-domain`、`检测域名`、`域名可达性`
- **描述**: 通过多地区节点发起 HTTP 请求，统计各节点成功率，判断域名在国内各运营商网络下的可达性。成功率 < 70% 提示可能存在访问异常。

## 依赖安装

```bash
pip install selenium
# 需本机安装 Chrome + ChromeDriver（版本需匹配）
```

## 工作流

```bash
# 基础用法：检测国内三网（电信 + 联通 + 移动）
python3 scripts/check_domain.py <域名1> [域名2] ...

# 显示每个节点的详细结果
python3 scripts/check_domain.py <域名> -v

# 包含港澳台、海外节点
python3 scripts/check_domain.py <域名> --overseas

# 组合使用
python3 scripts/check_domain.py <域名> -v --overseas

# 自定义异常判定阈值（默认 0.7）
python3 scripts/check_domain.py <域名> --threshold 0.5
```

## 参数说明

| 参数 | 简写 | 说明 |
|------|------|------|
| `域名` | | 一个或多个，支持 `https://example.com` 或 `example.com` |
| `--verbose` | `-v` | 显示各节点 IP、HTTP 状态码 |
| `--overseas` | `-o` | 包含港澳台、海外节点（默认只检测国内三网） |
| `--threshold` | | 异常判定阈值，默认 `0.7` |

## 工作原理

1. 启动 Headless Chrome，打开第三方公开 HTTP 检测页面
2. 按需选择检测节点范围（国内三网 / 含海外）
3. 填入域名，触发检测任务
4. 实时收集各节点返回的 HTTP 检测结果
5. 统计 HTTP 200 成功率，低于阈值提示访问异常
6. 退出码 `1` 表示存在异常节点（可用于 CI/定时巡检）
